PROJECT NAME: MediFlow

HACKATHON:
Queue Cure '26

PROBLEM STATEMENT:
76% of India's clinics still use paper token systems. Patients have no visibility into wait times. Receptionists manage queues manually. Build a real-time clinic queue management system.

PROJECT GOAL:
A web application where:

* Receptionist can add patients and manage queue.
* Patients can see current token, tokens ahead, and estimated wait time.
* Doctors can view queue analytics.
* All screens update instantly when "Call Next" is clicked.

TOKEN LOGIC:

When patient added:

Rahul -> Token 101
Priya -> Token 102
Amit -> Token 103

Auto increment token numbers.

WAIT TIME FORMULA:

Estimated Wait =
(tokens ahead) × (average consultation time)

Example:

Current Token = 101
Patient Token = 104
Average Time = 10 min

Tokens Ahead = 3

Estimated Wait = 30 minutes
